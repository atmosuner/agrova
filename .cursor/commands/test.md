# /test - Test-Driven Development

Activate the **Test-Driven Development** skill to verify code works with comprehensive testing.

## When to Use
- After implementing features
- When you want to prove code correctness
- Building confidence in changes

## What It Does
Guides:
- **Red-Green-Refactor** - Write tests first, then code
- **Test Pyramid** - 80% unit, 15% integration, 5% E2E
- **Browser Testing** - Chrome DevTools for runtime data
- **Test Quality** - DAMP over DRY, Beyonce Rule

## Key Benefit
Tests are proof, not afterthought. Catch regressions early.
