# /plan - Planning & Task Breakdown

Activate the **Planning & Task Breakdown** skill to decompose specs into small, verifiable tasks.

## When to Use
- After writing a spec
- When you need to estimate work
- Breaking complex features into manageable units

## What It Does
Creates verifiable tasks with:
- **Acceptance Criteria** - Clear definition of done
- **Dependencies** - Task ordering
- **Estimates** - Relative effort sizing
- **One Task = 1-2 hours** - Small, reviewable units

## Key Benefit
Smaller tasks = faster reviews, easier testing, quicker feedback.
