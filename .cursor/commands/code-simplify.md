# /code-simplify - Code Simplification

Activate the **Code Simplification** skill to reduce complexity while preserving behavior.

## When to Use
- Code works but is harder to read than it should be
- Need to reduce maintenance burden
- Before major refactors

## What It Does
Guides:
- **Chesterton's Fence** - Understand before changing
- **Rule of 500** - Functions under 500 lines, classes under 500 lines
- **Clarity Over Cleverness** - Simple > Smart
- **Preserve Exact Behavior** - No functionality changes

## Key Benefit
Clearer code is cheaper to maintain and debug.
