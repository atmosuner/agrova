# /spec - Spec-Driven Development

Activate the **Spec-Driven Development** skill to write a comprehensive Product Requirements Document before any code.

## When to Use
- Starting a new feature
- Beginning a significant change
- Defining project scope and constraints

## What It Does
Guides you through writing a structured PRD covering:
- **Objectives** - Why this matters
- **Signals** - How to measure success
- **Commands** - User interactions or API endpoints
- **Structure** - Architecture and data models
- **Code Style** - Specific conventions
- **Testing** - Test strategy and coverage targets
- **Boundaries** - What's explicitly excluded

## Key Benefit
Catch 80% of issues before writing code, preventing costly rework.
